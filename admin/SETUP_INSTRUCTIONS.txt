📋 PREREQUISITES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ Node.js (v14 or higher) - Download from https://nodejs.org/
✓ npm (comes with Node.js)
✓ A code editor (VS Code recommended)
✓ A web browser (Chrome/Firefox/Edge)


🚀 STEP 1: BACKEND SETUP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Open Terminal/Command Prompt

2. Navigate to backend folder:
   cd backend

3. Install dependencies:
   npm install

4. Create .env file:
   - Copy .env.example to .env
   - Windows: copy .env.example .env
   - Mac/Linux: cp .env.example .env

5. Start the backend server:
   npm start

6. You should see:
   🚀 Server running on port 5000
   📝 In-memory storage active - data will reset on restart

✅ Backend is now running on http://localhost:5000


🎨 STEP 2: FRONTEND SETUP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Open a NEW Terminal/Command Prompt (keep backend running)

2. Navigate to frontend folder:
   cd frontend

3. Install dependencies:
   npm install

4. Start the frontend development server:
   npm start

5. Browser should automatically open to http://localhost:3000

✅ Frontend is now running on http://localhost:3000


🎯 STEP 3: FIRST TIME USE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. CREATE COMPANY & ADMIN ACCOUNT
   ────────────────────────────────────────────────────────────────────────────
   - Go to http://localhost:3000/signup
   - Fill in the signup form:
     * Company Name: Your Company
     * Country: Select from dropdown
     * Currency: Auto-filled
     * Admin Name: Your Name
     * Admin Email: admin@company.com
     * Password: YourPassword123
     * Confirm Password: YourPassword123
   - Click "Sign Up"
   - You'll be logged in automatically

2. ADD USERS
   ────────────────────────────────────────────────────────────────────────────
   - Click "Add User" button
   - Fill in user details:
     * Name: Employee Name
     * Email: employee@company.com
     * Role: Employee or Manager
     * Manager: Select if applicable
   - Click "Add User"
   - ⚠️ IMPORTANT: Check the BACKEND CONSOLE for the generated password!
   - Password will be displayed like:
     
     =================================
     📧 EMAIL NOTIFICATION (STUB)
     =================================
     To: employee@company.com
     Temporary Password: Abc123XYZ!@#
     =================================

3. CONFIGURE APPROVAL RULES
   ────────────────────────────────────────────────────────────────────────────
   - Click "Approval Rules" in navigation
   - Select an employee from dropdown
   - Configure their approval workflow:
     * Manager: Select their manager
     * Approvers: Check users who can approve
     * Is Manager Approver: Check if needed
     * Required Approver: Check if mandatory
     * Approval Type: Sequential or Parallel
     * Minimum Approval %: Set 0-100
   - Click "Save Approval Rule"


📝 IMPORTANT NOTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️  DATA STORAGE
    All data is stored IN-MEMORY only. When you restart the backend server,
    ALL DATA WILL BE LOST. This is intentional for development/demo purposes.

📧  EMAIL NOTIFICATIONS
    Emails are NOT actually sent. Passwords are logged to the backend console.
    Always check the backend terminal for generated passwords.

🔒  SECURITY
    The .env file contains the JWT secret. In production, use a strong secret
    and never commit .env to version control.

🌐  PORTS
    - Backend: http://localhost:5000
    - Frontend: http://localhost:3000
    - Make sure these ports are not in use by other applications


🐛 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: "Port 5000 already in use"
Solution: Kill the process using port 5000 or change PORT in .env

Problem: "npm install fails"
Solution: Delete node_modules folder and package-lock.json, then run npm install again

Problem: "Cannot connect to backend"
Solution: Ensure backend is running on port 5000 and check browser console

Problem: "Data disappeared"
Solution: This is expected! Data resets when backend restarts. Signup again.

Problem: "Can't find generated password"
Solution: Look at the BACKEND terminal/console, not the frontend
